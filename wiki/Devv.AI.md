# Devv.AI

## 一句話定位

**3-4 人編制跑完 AI 搜尋引擎 → Coding Agent → Agent 管理平台三階段的極小新創樣本**。

### 團隊規模特色

- 始終維持 **3-4 人核心**
- Top 4 包攬 97.4% commits
- 對比通常「草根社群專案」外部貢獻佔多數——這是典型「正規軍團隊外流的 OSS」

## 三階段產品演化

### 2023：Devv 1.0——AI 搜尋引擎

- **地點**：上海
- **定位**：**開發者專屬 AI 搜尋引擎**
- **護城河**：自建垂直索引（Index）——專門爬官方 Docs、Stack Overflow、GitHub 程式碼，用 RAG 結合 LLM 回應
- **與通用 AI 差異**：不靠 ChatGPT 瞎掰，基於真實技術文件生成解答
- **起源**：Jiayuan 在動工前深訪 50 位 Developer，鎖定「通用 AI 吐垃圾程式碼」痛點
- **成長**：純靠口碑，幾個月內用戶突破 50 萬；2024 年中 ProductHunt 打榜時超過 60 萬
- **終局**：**因 GFW（防火長城）限制專案訪問，一夜核心盤歸零**
- **商業紀錄**：早期 3 人編制達到 $30K MRR

### 2025：Devv 2.0——Coding Agent

- **轉型動機**：Jiayuan 在 2025 年 LinkedIn 長文解釋——Agent 革命爆發後，「搜尋答案」的解法太慢，市場剛需變成「**直接生成並交付（Generate & Deliver）**」
- **定位**：第一個幫你交付 AI 產品的 Coding Agent
- **目標客群擴張**：從純後端工程師 → PM、Designer、無技術背景 Founder

### 2026：[[Multica]]——Agent 管理平台

- **定位**：管理多個 Coding Agent 的工作流
- **切入層**：工具鏈更底層的 Infra

## 戰略軌跡總結

> 2023 搜尋答案 → 2025 生成產品 → 2026 管理多 Agent

**每一步都往工具鏈更底層的 Infra 扎，扛起這條線的只有 3-4 個人**。

## 觀察

### 反脆弱性

GFW 讓 Devv 1.0 核心盤歸零——但團隊沒消失，反而藉 Agent 革命跳到下一層。**把「政策風險」當作強迫轉型的訊號**。

### 小團隊 + Agent 的放大器樣本

Multica 的 Slogan「兩個工程師帶一群 Agent 能動得像 20 個人」——實際上就是 Devv 團隊自身的**實戰體悟**，不是行銷話術。
