## 三大宏觀主題

### 主題 1：「自我校正」是知識庫的核心能力

W18 最重要的事不是新增了什麼，而是**主動發現並撤回**了 W17 期間累積的錯誤論述：

- **W17 期間記錄**：「ast-grep 連十日精選」「歷來最長紀錄」「擴散 S 曲線」「anchor skill 候選」
- **W18 校正發現（5/1）**：全部為 `fetch-skills-picks.sh` fallback 假象——skills.sh 04-19 起停發新 pick
- **校正動作**：
  1. [[src-skills-picks-2026-04]] 加入 `⚠️ 校正聲明` + 把虛構推論標為「保留作為紀錄」
  2. [[src-skill-ast-grep]] 修正「skills.sh 持續精選」為「2026-04-18 唯一一次精選」
  3. fetcher 修改：`is_fresh_pick: false` 明確標記
  4. 新增 [[src-skills-trending-2026-05]] 作為 organic 流行度主追蹤替代

> **教訓**：抓取腳本的**靜默 fallback 會製造資料假象**——LLM ingest 時若直接相信 raw，會把同一筆誤讀為「連續紀錄」並衍生虛構論述。已寫入 user memory（feedback_silent_fallback_creates_phantom）。

延伸事件：W18 末的 [[Inference.sh]] 改名事件——repo URL 變動造成 [[skills.sh]] 重新統計，**install 數單日 −7,000 不是真實流失**——這也屬於「資料管道層級」的解讀陷阱，需要持續校正視角。

### 主題 2：Agent 抽象的進化——Model + Body + Harness

W18 的論述產出由 [[src-agent-model-body-harness]]（4/29）牽頭，把 agent 的內部結構推進一步：

- **Model**（大腦） — intelligence 本身
- **Body**（手腳） — tool use、filesystem、bash、sandbox、browser、MCP
- **Harness**（約束） — CLAUDE.md、hooks、planner-evaluator、sub-agent、back-pressure

這個三層拆解解釋了**「同樣 Opus 4.6，OpenClaw vs [[Claude Code]] 體感差很多」**的根因。延伸至：

- [[src-bug-hunter]]（5/4 ingest）— **adversarial multi-agent + auto-fix on safe branch**——明顯運用「Body 層」的能力（write to safe branch）+「Harness 層」的對抗性協作
- [[src-Code Review已死]] 的 Swiss-cheese 五層信任 — 屬於「Harness 層」的設計
- agent VM 浪潮（PH 5/1-5/3：Manus Cloud / Huddle01 VMs）——**「Body 層的雲端化」**走向獨立產品市場

### 主題 3：AI agent 基礎設施的市場成形

W18 PH 連續 3 日 agent 基礎設施產品上榜：

| 日期 | 產品 | 類別 |
|------|------|------|
| 5/1 | Postiz / Buda | agent SaaS（給 agent 用的工具） |
| 5/2 | Manus Cloud Computer / explainx ai | agent VM + skill 市集 |
| 5/3 | Huddle01 VMs / PandaProbe / Rosentic | agent VM + engineering 平台 + agent conflict detection |

這些產品的共同模式：**「agent 不只是使用者的工具，本身需要專屬基礎設施（VM、storage、conflict detection）」**。對應 wiki 已有 [[Multica]]（Devv.AI 的 agent 平台）— W18 期間多了 5+ 個競爭對手進入 PH 視野。

外部 [[Anthropic]] 同步推 Creative Work 8 connector（4/28），標誌**「agent 基礎設施 + 應用整合」雙向擴張同時發生**。
