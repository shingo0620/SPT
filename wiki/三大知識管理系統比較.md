# 三大知識管理系統比較

三位科技領袖的 AI 知識管理方法論並排比較。

## 總覽

| 維度 | [[LLM Wiki]] | [[QMD]] | [[GBrain]] |
|------|-------------|---------|------------|
| 提出者 | [[Andrej Karpathy]] | [[Tobi Lütke]] | [[Garry Tan]] |
| 隱喻 | 羅盤 | 望遠鏡 | 營運系統 |
| 核心功能 | 編纂與結構化 | 極致檢索 | 全生命週期營運 |
| 資料規模 | 數十～數百頁 | 不限（本地） | 萬級頁面 |
| 儲存方式 | Markdown 檔案 | 本地檔案 | Postgres + pgvector |
| 檢索方式 | index.md + wikilink | BM25 + 向量 + 重排序 | 關鍵字 + 向量 + RRF |
| 雲端依賴 | 無（LLM API 除外） | 無 | 有（資料庫 + 嵌入） |
| Agent 角色 | 維護 wiki | 可透過 MCP 呼叫 | 讀寫 + enrichment + 夢境維護 |
| 知識更新 | 手動觸發 ingest | 被動（搜尋時） | 主動（訊號驅動） |

## 哲學差異

| 系統 | 核心信念 |
|------|----------|
| LLM Wiki | 好編纂勝過好檢索 |
| QMD | 找得到比寫得好更重要 |
| GBrain | 知識要能自動變聰明 |

## 適用光譜

```
個人深讀 ←————————————————————→ 企業級營運
  LLM Wiki          QMD              GBrain
  （編纂）        （檢索）          （管線）
```

- **起步階段**：LLM Wiki 最輕量，立即可用
- **檔案量暴增**：QMD 補上檢索能力
- **需要自動化與規模**：GBrain 模式的資料庫與管線

## 共通原則

三者雖然路徑不同，但共享以下設計原則：

1. **不可變原始資料**：原始來源不被修改
2. **結構化沉澱**：知識不是留在聊天紀錄裡消失
3. **Agent 可讀寫**：AI 不只是搜尋工具，而是知識管理的協作者
