# 向三位大師學 AI 筆記

## 摘要

本文比較三位科技領袖的 AI 知識管理方法論，各自代表不同規模與哲學：

- [[Andrej Karpathy]]（羅盤）：用 [[LLM Wiki]] 讓知識像 wiki 一樣持續生長，而非每次從零檢索。
- [[Tobi Lütke]]（望遠鏡）：用 [[QMD]] 在本地做到極致檢索，隱私優先、零雲端。
- [[Garry Tan]]（營運系統）：用 [[GBrain]] 把個人知識庫當公司營運，Postgres + pgvector 支撐萬級頁面。

## 與 Roger's Letter 系統的對照

作者自身的系統以投資研究為核心，在 Cursor 中以 skills 組織研究管線，以儀表板呈現公司與產業地圖。與三位大師的對齊：

| 大師提問 | Roger's Letter 回應 |
|----------|---------------------|
| Karpathy：知識有沒有在長？ | 有，長在公司與產業的研究資產裡 |
| Tobi：找不找得到？ | 儀表板與 API 內建結構化查找；極致檢索可加 QMD |
| Garry：能不能每天自動變聰明？ | skills + 管線 + 同步讓研究流程複利 |

## 關鍵洞見

- 「好編纂勝過好檢索」——知識要被寫進結構，不是每次從零拼湊
- 三種系統對應三種規模：個人深讀 → 本地全域搜尋 → 企業級營運
- 共通原則：不可變原始資料、結構化沉澱、Agent 可讀寫
