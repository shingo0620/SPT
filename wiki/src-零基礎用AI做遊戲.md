# 零基礎用 AI 做遊戲——從陽春到華麗的全過程

## 來源資訊

- 頻道：Debug 土撥鼠
- 連結：https://www.youtube.com/watch?v=SeiG9ZrLcBc

## 摘要

一位零程式基礎的創作者，用 Claude Code 搭配遊戲引擎製作了 5 款遊戲，並示範不花一分錢、全程用免費 AI 工具生成美術與音樂的完整工作流。核心證明：不會寫程式、沒有美術基底，也能做出可玩且有完整美術的遊戲。

## 遊戲引擎選擇

選擇標準：
1. 一切可透過程式碼生成（方便 AI 操作）
2. GUI 有 MCP 整合更好

| 引擎 | 語言 | 適合 | 代表作 |
|------|------|------|--------|
| [[Godot]] | GDScript（類 Python） | 2D/3D 通用 | Brotato、Dome Keeper、Backpack Battles |
| Love2D | Lua | 2D 專精 | Balatro |

## 五款實作遊戲

| # | 遊戲 | 複雜度 | 引擎 | 開發時間 |
|---|------|--------|------|----------|
| 1 | Candy Crush 複製 | 低 | Godot | < 10 分鐘 |
| 2 | 台灣麻將 | 中（規則複雜） | Godot | < 1 小時 |
| 3 | Vampire Survivors 類 | 高（動畫、技能） | Godot | 多輪迭代 |
| 4 | 原創派對遊戲 | 高（多人、道具） | Love2D | 大量對話 |
| 5 | 3D 煮過頭復刻 | 高（3D） | Godot | 多輪迭代 |

## 迭代方法論

核心原則：**看到什麼醜就修什麼**

```
AI 產出原型（機制正確但醜）
  → 角色醜？換像素風
    → 地板醜？換地板風格
      → 加音效 → 修版面 → …
        → 最終成品
```

與 [[遊戲開發流程]] 的原型驗證階段一致：先用佔位符跑通核心玩法，再逐步提升視覺品質。

## 免費 AI 美術工作流

完整鏈路，零花費：

```
Claude Code（產生所有素材 prompt）
  → Gemini Banana 2（生成圖片：角色、背景、道具）
    → Grok（圖片轉影片：角色動畫）
      → Blender（影片去背：綠幕移除）
        → Suno（生成遊戲配樂）
```

### 實用技巧

- **綠幕技術**：圖片 prompt 結尾加 `#00FF00 background`，方便後續去背
- **浮水印處理**：Gemini 生成的圖片右下角有星形浮水印，用綠色方塊覆蓋
- **Sprite Sheet**：Gemini 可直接生成分解動畫幀，裁切後即可使用
- **Blender 去背**：Video Edit → 加 Keying Modifier → 用滴管選綠色 → Render Animation → 輸出 RGBA PNG 序列
- **Suno 版權**：免費版生成的音樂沒有商用權利
