# 遊戲開發流程

遊戲開發的通用簡化模型，適用於單機與網遊、不同地區與公司，差異在細節而非骨架。

## 五大階段

```
立項 → 前期製作 → 正式開發 → 測試 → 製作後期
                    (60%-80%)
```

### 1. 立項（Concept / Pitch）

**目標**：調研市場、提出概念、取得啟動資金。

推介（pitch）= 向發行商/投資人展示專案構想，是第一個里程碑。需要包含：
- 目標賽道與用戶畫像
- 核心玩法與差異化
- 團隊經驗與可執行性
- 嚴謹的市場數據（競品銷量、社區活躍度、Steam 指標）

資金來源：

| 模式 | 優點 | 風險 |
|------|------|------|
| 發行商簽約 | 穩定資金、行銷渠道 | 分成低、創作受限 |
| 眾籌 | 保留自主權、早期驗證市場 | 資金可能不足、社區壓力 |
| 股權投資 | 大額資金、可保留創作權 | 股權稀釋 |

### 2. 前期製作（Pre-production）

**最重要的階段——決定專案生死。**

必須驗證兩件事：
1. **核心玩法可行性**：玩法循環是否成立？體驗是否有趣？
2. **技術可行性**：引擎與工具鏈能否支撐設計需求？

驗證層次：原型 → 首次可玩版 → [[垂直切片]]

產出物：
- GDD（Game Design Document，遊戲設計文檔）
- TDD（Technical Design Document，技術設計文檔）

與 [[MDA 框架]] 的關係：前期製作本質上是在驗證「機制 → 動態 → 美感」的因果鏈是否成立。

### 3. 正式開發（Production）

美術/策劃/程式「鐵三角」全力協作，佔總開發時間 60%–80%。

**三大噩夢**：

1. **團隊協作困難**：人數越多管理成本指數級上升
2. **技術債務爆發**：截止日壓力下的臨時方案集中爆發
3. **玩法中途推翻**：高層變卦、核心人員離職、市場風向突變

### 4. 測試（Testing）

```
Alpha → 內部測試 → Beta → EA（可選）→ 金盤
```

- **Alpha**：核心功能完成，流程跑通
- **Beta**：外部玩家測試，接近真實環境
- **搶先體驗（EA）**：以零售價發售，適合高重玩性類型
- **金盤（Gold Master）**：開發完成，交付工廠或數位商店

### 5. 製作後期（Post-launch）

- 單機：Bug 修復、平衡性補丁、DLC
- 網遊：持續營運循環（版本規劃 → 開發 → 測試 → 上線）

## 發行模式演變

```
傳統：發行商代理（資金+行銷+渠道）
  ↓
過渡：開發發行一體化（育碧、EA、卡普空、拉瑞安）
  ↓
現代：開發者直面玩家（數位商店+社群行銷）
```

## 與 [[遊戲發想]] 的關係

發想是立項階段的核心動作。發想階段是否產出「具體方向」，直接影響後續所有階段的效率。
