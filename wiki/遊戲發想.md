# 遊戲發想

將靈感轉化為具體開發方向的過程。在 [[遊戲開發流程]] 中屬於立項階段的核心動作。

## 核心原則：具體方向

**失敗專案的共通點：發想階段未提出具體方向。**

- 「具體」：聽到名詞，腦中馬上有畫面（港漫風格、8-bit 像素風、回合制 JRPG）
- 「不具體」：每人想像不同（很特別、很夢幻、中國風）
- 結果：不具體 → 試試看 → 重做 → 迴圈 → 只有最後 1/4 內容上市 → 賠錢

## 心智圖四問法

使用心智圖（Mind Map）圍繞遊戲中心概念，逐層輻射回答四個問題：

### 1. 為什麼想做這款遊戲？

記錄初心——支撐漫長開發的動力。務必誠實。為什麼是遊戲而不是小說、影片？

### 2. 這是一款什麼樣的遊戲？

- 遊戲類型（要具體到子類型）
- 預計遊戲時長
- 上市平台
- 目標客群
- 開發工具

### 3. 遊戲風格是什麼？

三個維度必須檢查一致性：
- **視覺風格**
- **劇情風格**
- **玩法風格**

衝突不一定不好，但必須是**刻意的**選擇。

### 4. 故事或背景設定

- 故事簡介（一兩句話）
- 地理背景（決定地圖範圍 → 提前控制工作量）
- 年代（影響人物言行與視覺）
- 世界觀

> 劇情導向遊戲重點在故事；玩法導向遊戲把玩法獨立拉出來問，劇情配合玩法。

## 交叉驗證

畫完心智圖後的必要檢查：

```
目標客群 ←→ 遊戲風格
開發工具 ←→ 遊戲風格
初衷     ←→ 目標客群
故事設定 ←→ 遊戲風格
```

任何不一致都需要回頭調整，除非衝突是刻意設計。

## 與 [[MDA 框架]] 的互補

- 發想定義**目標體驗**（美感層）
- MDA 提供從美感逆推到機制的路徑
- 發想回答「做什麼遊戲」；MDA 回答「怎麼讓它好玩」

## 團隊規模差異

| 規模 | 注意事項 |
|------|----------|
| 一人獨立 | 排除超出能力範圍的方向；[[AI 輔助遊戲開發]] 可大幅擴展一人可及的範圍 |
| 中小型團隊 | 集體創作，不是個人作品；量力而為 |
| 大型團隊 | 即使 AAA 也因餅畫太大而暴死 |
