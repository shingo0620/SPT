# 秦无邪 OvO — 我似乎已经看见了，游戏引擎的未来！

> YouTube：https://www.youtube.com/watch?v=aNWPR4miOvY
> 作者：秦无邪 OvO（自稱「五鞋」，遊戲製作人）
> 發布日期：2026-03-20
> 影片長度：~8 分鐘

## 核心主張

從**遊戲引擎的歷史 → 商用 vs 自研 → Unity 風險 → 未來預測**——預測未來遊戲引擎將是「**人 + AI 代理共同使用**」的軟體形態，更多 API 留給 AI，人只把控大方向。並把現在的遊戲產業類比為**文藝復興早期**——AI 解放生產力後，「決定做什麼比怎麼做更重要的時代」終於要來了。

## 1. 遊戲引擎的本質與分類

### 鍋灶台比喻

| 角色 | 對應 |
|------|------|
| **遊戲引擎** | 鍋與灶台（基礎設施） |
| **美術 / 音樂 / 程式碼** | 食材 |
| **遊戲設計** | 菜譜 |
| **遊戲開發者** | 廚師 |
| **完成的遊戲** | 端上的菜 |
| **遊戲類型** | 川菜、粵菜、江浙菜（不同口味的菜系） |

### 三種引擎類型

| 類型 | 範例 | 經典案例 |
|------|------|---------|
| **有 GUI 商用 / 開源** | Unity、Unreal、[[Godot]]、GameMaker、Cocos | 多數主流遊戲 |
| **無 GUI 代碼庫式** | MonoGame、FNA、Love2D、Raylib、LibGDX | 星露谷物語（MonoGame）、**蔚藍**（FNA）、**小丑牌 Balatro**（Love2D）、**殺戮尖塔**（LibGDX） |
| **垂直領域** | RPG Maker | 《巨月球 To the Moon》 |

## 2. 商用 vs 自研引擎

### 大廠為何自研引擎

不是炫技，而是**商用引擎難覆蓋的需求**：超大規模在線、特殊渲染管線、高度自定義服務器架構。

| 公司 | 自研引擎 | 代表作 |
|------|---------|-------|
| 騰訊 | Quicksaver | 天涯明月刀 |
| 網易 | NeoX | 第五人格、蛋仔派對（同引擎不同風格） |
| 米哈遊 | Unity + 源碼授權多年改造 | （某種意義上等同自研） |

### 自研優勢

**完全自主可控的渲染管線**——同一引擎可捏成第五人格的哥特童話風 + 蛋仔派對的卡通 Q 版風格。中小團隊用商用引擎沒源碼改權，只能透過修 shader 和後處理來差異化。

### Unity 風險事件清單

| 年份 | 事件 |
|------|------|
| 2023 | 運行時收費事件 |
| 2025 | 收割大客戶事件 |
| 2026 | 國際版商店切割事件，導致開發者損失慘重 |

→ 商用引擎的政策變動是中小團隊不可控的結構性風險。

### 自研缺點

- **招人困難**——市場上引擎開發者稀缺
- **更新困難**——已上線遊戲換引擎「像給住滿人的摩天大樓換地基」
- 引用 Naughty Dog GDC 演講：「**遊戲引擎對開發者既是工具也是限制**」

## 3. 第五人格的引擎升級案例

8 年運營後玩家最常見抱怨：幀率不穩。最近一次 NeoX 引擎更新重點為**性能優化**：

1. **多線程技術**：原本主循環全跑在主線程 → 拆分多線程做圖形渲染，降低主線程負擔
2. **Shader 編譯線程**：把「大哥的活」分一部分給「閒著的小弟」
3. **整體成果**：戰鬥內平均幀率達穩定狀態

→ 案例證明引擎升級對長期運營遊戲的關鍵價值，但難度極高。

## 4. 核心預測：遊戲引擎的未來

### 趨勢觀察

> **「過去十幾年遊戲引擎一直是給人用的，所以 GUI 編輯器是否好用特別重要。但現在新的趨勢出現：越來越多軟件開始被 AI 代理所使用，而不是人類。」**

### MCP 的角色

引用《火影忍者》傀儡師比喻：
- **傀儡師** = AI 代理
- **查克拉線** = MCP（Model Context Protocol）
- **傀儡** = 軟體

> 「以前我們人是傀儡師、軟體是傀儡。現在 AI 代理變成傀儡師，幫我們控制軟體。」

### 作者個人實踐

> **「在 Claude Code 出來之前，我大多數操作都在各家遊戲引擎裡手動進行。現在基本上都是讓 AI 大模型幫我寫好功能，我只需要負責點擊運行測試。甚至運行和測試現在都可以透過 MCP 來幫助。」**

### 預測

**未來遊戲引擎 = 人 + AI 代理共同使用的軟體形態**：
- **更多 API 留給 AI**
- **人只負責把控大方向的接口**
- MCP 也只是個過渡產物

> **「決定做什麼比怎麼做要重要得多的時代終於要來了。」**

## 5. 遊戲文藝復興類比

| 文藝復興時期 | 當代遊戲產業 |
|------------|------------|
| 造紙技術發展 | 遊戲引擎越來越成熟 |
| 紙張大規模傳播 | 開發工具越來越完善 |
| 文學、藝術、思想爆發 | AI 強大輔助 → 越來越多人參與遊戲創作 |

> **「遊戲可能才剛迎來屬於它自己的文藝復興。」**
