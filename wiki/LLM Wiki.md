# LLM Wiki

由 [[Andrej Karpathy]] 提出的個人知識管理模式。不是軟體，而是一種工作模式——用 LLM 逐步建立並維護一個持久存在的 wiki。

## 核心理念

**好編纂勝過好檢索**——知識要在多次對話與多份來源中被寫進結構，而不是每次都從零拼湊。

> wiki 是一個持續存在、會不斷複利增長的成果物。交叉引用已經存在，矛盾也早已被標記出來。

人類負責：挑選來源、引導分析、提出好問題。
LLM 負責：摘要、交叉引用、歸檔、整理與維護。

## 三層架構

1. **Raw sources**（不可變真實來源）：你精心挑選的文件，LLM 只讀不改
2. **Wiki**（LLM 維護的 Markdown 目錄）：摘要、實體頁、概念頁、比較頁、綜整頁
3. **Schema**（CLAUDE.md / AGENTS.md）：約束 LLM 行為的設定檔，最重要的設定——讓 LLM 成為有紀律的 wiki 維護者

## 三種操作

- **Ingest**：匯入來源。LLM 讀取、討論重點、寫摘要、更新實體與概念頁。單一來源可能影響 10-15 頁。偏好一次一個、持續參與。
- **Query**：查詢 wiki。LLM 搜尋相關頁面、整理附帶引用的回答。**關鍵洞見：好的回答可回存 wiki 成為新頁面——探索也產生複利。**
- **Lint**：健康檢查。矛盾、過時主張、孤立頁面、缺漏概念、缺少交叉引用、資料缺口。

## 索引與日誌

- **index.md**：內容導向的總目錄，按類別組織。LLM 回答問題時先讀索引找相關頁面。
- **log.md**：時間導向的操作紀錄，僅追加不覆寫。

在中等規模下（約 100 來源、數百頁），索引 + 連結的導航方式意外地非常有效，避免了建立嵌入式 RAG 基礎設施的需求。

## 為什麼取代 RAG

傳統 RAG（NotebookLM、ChatGPT 檔案上傳等）的痛點：
- 需要向量資料庫與 Embedding 模型，技術門檻高
- 每次查詢都從頭重新發現知識，沒有累積性
- 細微問題需綜合多份文件時，每次都得重新拼湊

LLM Wiki 的優勢：
- 等同建立 Graph 資料庫，但只需訂閱 Claude
- 知識編譯一次，之後持續更新
- 可發佈成網頁供團隊查詢
- Coding Agent 大幅消滅了 RAG 過去的複雜性

## 為什麼有效

人們放棄 wiki 是因為維護成本增長超過價值——更新交叉引用、保持摘要最新、標記新資料推翻舊主張、跨數十頁維持一致性。

LLM 不會覺得無聊、不會忘記更新交叉引用，能一次處理 15 個檔案。維護成本接近零，wiki 因此能持續維護。

## 實作方式

透過 Claude Code（或 OpenAI Codex、OpenCode 等）驅動，五步即可建立：

1. 建立專案資料夾
2. 建立 `raw/` 子目錄
3. 放入參考資料（支援 Word、Excel、PDF、MD、TXT）
4. 貼入 LLM Wiki Prompt 範本
5. 加上客製化指令，Agent 自動運作

持續更新：新檔案放入 `raw/`，指示 Agent 整理即可。

## Obsidian 工具整合

- **Web Clipper**：瀏覽器擴充套件，網頁轉 Markdown 匯入 raw/
- **圖片本地化**：附件資料夾設為 `raw/assets/`，綁定快捷鍵下載內嵌圖片
- **Graph View**：觀察 wiki 結構，辨識樞紐與孤點
- **Marp**：從 wiki 內容直接生成 Markdown 簡報
- **Dataview**：對 frontmatter 執行查詢，動態產生表格與清單

比喻：Obsidian 是 IDE；LLM 是程式設計師；wiki 是程式碼庫。

## 可選擴展：CLI 搜尋工具

規模擴大後可加入搜尋引擎（如 [[QMD]]），支援混合式 BM25 / 向量搜尋與 LLM 重排序。小規模下 index.md 就夠用。

## 歷史脈絡：Memex

精神上承接 Vannevar Bush 於 1945 年提出的 Memex——一種個人化、經過策展的知識儲存系統，文件之間有聯想式路徑相連。Bush 的願景比後來的網路更接近 LLM Wiki：私人的、主動策展的，文件之間的關聯與文件本身同樣有價值。他當年無法解決的維護問題，現在交給 LLM。

## 適用場景

- 個人：目標、健康、心理狀態、自我成長追蹤
- 研究：論文與報告的持續整合
- 讀書：角色、主題、情節線與關聯（類比 Tolkien Gateway 粉絲 wiki）
- 商業團隊：Slack、會議逐字稿、專案文件、客戶通話的內部 wiki
- 其他：競品分析、盡職調查、旅遊規劃、課程筆記
